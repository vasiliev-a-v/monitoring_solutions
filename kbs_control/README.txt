Программа проверки состояния базовых станций типа БС6 и БС7
на контроллере базовых станций (КБС) "Гудвин"
Программа опрашивает КБС раз в минуту:
1. Скачивает с КБС текущий лог (журнал), проверяет БС на наличие "off",
2. если есть "off", значит базовая станция отваливалась
3. проверяет есть ли далее в логе после "off" строка с "on"
4. если "on" нету, то значит базовая станция зависла,
5. включает счетчик "терпения" на соответствующую базовую станцию
6. пишет в свой журнал дату и номер отвалившейся базовой станции,
7. перезапускает коммутационный процесс g2
8. сбрасывает счетчики БС и поток E1_АТС
9. ставится на паузу - в следующий раз начинает проверять не ранее чем через 5 минут
10. программа запускается стартовым процессом.
- папки ks11-ks14 - это шаблоны КБС для примера. На основе их надо создавать свои КБС
- перед запуском необходимо скопировать и отредактировать папки ks11-ks14, записав настройки для подключения к КБС: IP, название
- для отображения своей КБС имя папки необходимо записать в index.htm, заменив ks11-ks14 на имя папки своей КБС 
- скачанные с КБС логи расположены во временной папке /tmp/ks11-ks14
- для установки необходимо настроить apache разрешить на исполнение файлы *.sh и *.bash  

Функционал веб-интерфейса:
- вводить/выводить БС из мониторинга
- перезапускать процесс g2 внутри КБС вручную
- перезагружать КБС
- включать КБС в режим пассивного наблюдения (без автоматических перезапусков)
- приостанавливать и перезапускать службу наблюдения
- смотреть журналы: лог КБС, журнал перезапусков БС, журнал процесса kbs_control.sh

Файлы:
kbs_control.sh - фоновые службы контроля за КБС Гудвин (запускается несколько экземпляров по количеству наблюдаемых КБС)
kbs_log.bash   - отображает журнал скачанный с КБС
error_log.bash - журнал отображает время отключения базовых станций на КБС, перезапуска процесса g2 и перезагрузки КБС
process_log.bash - отображает процессы в kbs_control.sh фоновой службе контроля за КБС Гудвин
start.sh - стартовый процесс. Запускает экземпляры kbs_control.sh из каталогов
web-kbs.bash - ВЕБ-ИНТЕРФЕЙС / контролирует управляет экземплярами КБС Гудвин
index.htm - веб-страница с фреймами для контроля у управления несколькими КБС

common_config.txt - конфигурационный файл подгружаемый всеми экземплярами kbs_control.sh
kbs_config.txt - прописываются (и в последствии сохраняются) базовые станции и необходимость их контролировать (on/off)

Примечание:
1. перед запуском программы 
для согласования методов шифрования при подключении к старым КБС необходимо записать в файле ~./ssh/config
---
#~ Legacy changes
#~ Для подключения к древнему КБС Гудвин
KexAlgorithms diffie-hellman-group1-sha1
Ciphers aes128-cbc
---
И перезапустить ssh
sudo systemctl restart ssh.service
